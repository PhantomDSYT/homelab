# ADR-005: Adopt AdGuard Home as the Network DNS Resolver

**Status:** Accepted

**Date:** 2026-07-14

---

## Context

As the homelab expands, managing network traffic and understanding device behavior becomes increasingly important.

Traditional public DNS providers provide name resolution but offer limited visibility and control over DNS queries generated by devices on the network.

A self-hosted DNS solution would allow greater control over DNS traffic, provide network-wide filtering, and improve visibility into how devices communicate externally.

---

## Decision

Adopt AdGuard Home as the primary DNS filtering and management solution for the home network.

AdGuard Home will handle DNS requests from network devices and provide filtering, query visibility, and future DNS customization.

---

## Rationale

AdGuard Home was selected because it provides:

- Network-wide advertisement blocking.
- Visibility into DNS queries from connected devices.
- Control over filtering rules.
- Custom DNS rewrite capabilities for internal services.
- A self-hosted alternative to relying solely on third-party DNS providers.

Running DNS locally also provides a valuable learning opportunity around networking, name resolution, and infrastructure services.

---

## Alternatives Considered

### Public DNS Providers

Examples:
- Google DNS
- Cloudflare DNS

**Advantages**

- Simple setup.
- Highly available.
- No maintenance required.

**Disadvantages**

- Limited control.
- No visibility into local DNS traffic.
- No custom internal DNS records.

---

### Pi-hole

**Advantages**

- Popular self-hosted DNS solution.
- Large community.

**Disadvantages**

- AdGuard Home provided a feature set and interface that better aligned with current goals.

---

## Consequences

### Positive

- Network-wide DNS filtering.
- Better visibility into device activity.
- Internal service discovery through DNS rewrites.
- Greater control over network infrastructure.

### Negative / Trade-offs

- DNS becomes another service that requires maintenance.
- A failure of the DNS server can impact network name resolution.
- Requires proper monitoring and backup planning.

---

## Implementation Notes

Current deployment:

- AdGuard Home running in Docker.
- DNS management interface kept private.
- Future integration planned with Unbound for recursive DNS resolution.
